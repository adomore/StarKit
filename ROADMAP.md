## Phase 1 — MVP CLI (gate G1) — FR-1, FR-2, FR-3 (manual path), FR-4, FR-5 core, FR-8 minimal

- [x] **T1-1 `starkit-io`:** decode TIFF 8/16 + PNG + JPEG → linear f32 RGB; capture ICC bytes + EXIF; encode TIFF16 restoring both; atomic write path (the only write path). ICC backend decision → D-005.
  **AC = FR-1:** no-op run pixel-identical for TIFF16; truncated/corrupt input fails cleanly with no partial file; atomic-semantics test (kill between temp write and rename leaves target untouched).
  **Done** 2026-07-16. 46 tests (33 unit + 13 FR-1 acceptance). **No-op round-trip is pixel-identical on the real 4096² `basic-5k` fixture** — all 50.3M samples — not only on a toy image. Corrupt/truncated/non-image inputs each fail with the right typed variant and create no output file. Atomic path: temp beside target → fsync → rename, with failure-injection tests proving a failed write leaves the previous target byte-identical and leaves no debris; INV-3 checked by hashing the input before and after. Decisions: **D-029 resolves D-005 — no ICC backend at all** (working space is linear light in the source's *own* primaries; converting to a canonical space would clip wide-gamut data and break pixel-identity), curve round-trip proven exhaustively over all 65536 codes; **D-030** EXIF is preserved TIFF→TIFF (sub-IFD resolved and rewritten with correct offsets) but **dropped for JPEG/PNG→TIFF**, and GPS/Interop/MakerNote are never copied.
- [x] **T1-2 `starkit-core::background`:** tiled sigma-clipped background + noise map.
  **AC:** background RMS error within the tolerance documented at the test site on `basic-5k`; deterministic.
  **Done** 2026-07-16. 19 tests (15 unit + 4 AC on the real fixture). Measured on `basic-5k`: **background RMS error 0.19 ADU** against a ~300 ADU sky (0.06 %), tolerance 1.0 ADU — essentially the statistical floor (a 4096-sample tile median has a 0.215 ADU standard error), signed bias +0.13 ADU. **Noise map 11.10 ADU vs the generator's 11.02 (+0.79 %)**, tolerance 5 %. Clipping is load-bearing: the naive mean reads +5.54 ADU high from the 5000 stars, and the test asserts that naive bias exists so it cannot pass vacuously. Deterministic: bit-identical maps across runs. Two findings — the 3×3 grid median needed **clamped** edge padding (truncating the window biased every border tile toward the frame centre: 110.0 vs 104.9 on a ramp, i.e. a wrong background along all four edges of any gradient image), and the clip-bias correction is applied analytically (~4 % at 3σ, otherwise every downstream SNR would read optimistic). **D-031: the fixtures are linear but untagged, so `starkit-io` would mis-linearize them — flagged, must be decided at T1-3.**
- [x] **T1-3 `starkit-core::detect`:** threshold + local maxima, subpixel centroid, elliptical FWHM, deblending, saturation flags, tiering.
  **AC = FR-2 synthetic:** recall ≥ 98 % @ SNR ≥ 5, precision ≥ 99 %, median centroid error ≤ 0.3 px, median FWHM error ≤ 10 % on all applicable suites; **plus** oracle cross-check report (Δrecall ≤ 0.5 pp vs oracle on `basic-5k`; deviations beyond that require a DECISIONS entry).
  **Done** 2026-07-16. 22 tests (14 unit + 8 AC). **`basic-5k` meets all four bars: recall 99.94 % · precision 99.92 % · median centroid 0.098 px · median FWHM error 5.77 %** — better than the oracle on three of four (99.21 / 99.87 / 0.056 / 9.90). Graded through the real `starkit-io` decoder (D-032), against truth, with the FIXTURES.md matching rule. "Applicable suites" = `basic-5k` (D-033, owner ruling): the oracle reaches 98/99 on no other suite, so any wider reading is unachievable by its own reference. Others pinned as regression floors: `dense-core` 94.19 %/99.51 %, `saturated` 99.60 %/95.22 %, `pairs` 69.32 %/96.58 %, `nightscape-fg` 96.25 % recall. Oracle cross-check exceeds its 0.5 pp window at **Δ = 0.73 pp, by beating it** — logged per the AC (D-033). **`nightscape-fg` precision is 35.7 % and is T1-5's to fix (D-034)** — the D-019 silhouette effect; gating belongs in the single `gate` compositor, not in `detect`.

  Three bugs the tests caught, none of them what they first looked like: **(a)** the centroid window was 2.5 × FWHM = 8 px while `basic-5k` places stars a minimum of 8 px apart, so a neighbour on the rim dragged the centroid past the match radius — costing recall *and* precision at once (96.5 %/95.8 %, and a star at SNR 202 reading as "undetected"); a 1.2 × FWHM window fixed both to 99.9 %. **(b)** FWHM area integration was sub-pixel across rows but counted whole rows, so the bias oscillated with true FWHM (−13.6 % at 1.8 px, +6.5 % at 2.5 px) — a bias that changes sign indicts the estimator, not the profile; a flood fill on a ×8 supersampled grid took it to ≤0.5 %. **(c)** the matched filter was sum-1, which passes the local level through, so shot noise riding on the `saturated` suite's bright halos cleared a sky-derived threshold: 415 false positives against 500 real stars (precision 54.6 %). A zero-sum (mean-subtracted) kernel measures contrast instead — precision 95.2 %.
- [x] **T1-4 `starkit-core::mask` + export:** tiered feathered masks (radius = k × FWHM), saturated-halo extension; 16-bit grayscale export via `starkit-io`.
  **AC = FR-4:** golden mask hashes stable; pixel-alignment test vs source dimensions.
  **Done** 2026-07-16. 22 tests (16 unit + 6 AC). Golden hashes pinned over the **exported 16-bit codes** for `basic-5k`/`saturated`/`pairs`, verified to have teeth (a 1 % radius change fails them). Pixel alignment asserted for all four masks (large/medium/small/combined) on all five suites; export checked end-to-end through `starkit-io::encode_gray16` — 16-bit grayscale, no tone curve, full coverage lands on 65535. Every catalogued star on `basic-5k` is >50 % covered, and the mask covers <10 % of the frame (a selection of stars, not of sky). Profile choices in **D-035**: smoothstep feather (a linear ramp rings under a PS curve), union by maximum (coverage does not add), saturated halo ×2 (a clipped core's FWHM describes the hole, not the star).
  **Note:** FR-4's real bar — "photographer confirms the exported mask is usable as-is on ≥ 5 of her own images" — is **not** met and cannot be here: it needs the real corpus (D-027). Blocked at G1, with T1-10.
- [x] **T1-5 Sky-mask ingestion + gating:** import external grayscale mask; enforce INV-1 in the single compositor.
  **AC = FR-3 manual path:** invariant test proves pixels outside the sky mask are bit-identical, including on `nightscape-fg`.
  **Done** 2026-07-16. 22 tests (16 unit + 6 AC). **INV-1 proved on the real `nightscape-fg`**: 0 foreground pixels touched at zero tolerance, with an absurd modified plane so nothing can hide in rounding — and again with an all-NaN plane, since a buggy operation must not be able to put a NaN on the photographer's trees. Not vacuous: the gate opens on 0.5–30 % of the frame, asserted. Sky masks import via `starkit-io::decode_mask` (8/16-bit PNG/TIFF, grey or RGB, no tone curve — 50 % grey means half-selected); a mask sized for a different image is **refused, not resampled**. **D-034's debt paid: `nightscape-fg` precision 35.7 % → 99.91 %** (oracle 99.93 %) via `gate::restrict_to_sky` at a margin of `detect::filter_reach_px()` + 3 px — derived from the kernel, not guessed, so the two cannot drift. Costs recall 96.25 % → 88.97 % (oracle 91.63 %); D-036 explains why that trade runs the right way. Compositing **branches rather than blends** (D-036): a blend at a = 0 is not bit-identical — it turns `-0.0` into `+0.0` and carries NaN across a shut gate.
- [x] **T1-6 Reduction B (morphological):** masked erosion, N iterations.
  **AC:** outside-mask bit-identity; determinism; golden visual fixture.
  **Done** 2026-07-16. 17 tests (12 unit + 5 AC). **All three AC met on the real fixtures, through the full detect → mask → gate → reduce pipeline:** outside-gate bit-identity at zero tolerance on `basic-5k` *and* `nightscape-fg` (a reduction touches no nebulosity and no trees), determinism (bit-identical across runs), and a golden reduced-output hash pinned and verified to have teeth (a radius change fails it). This is the first *star operation*, so it is the first real exercise of the compositor rule — `morphological` erodes the whole plane and only `gate::composite` writes it back. Choices in **D-037**: disc element (a square erodes diamond-shaped stars), clamped edges (zero-pad would darken the border), NaN cannot win the minimum. The half-max shrink law is a **measured monotone trend**, not the continuous `R−r` ideal — the digital disc under-reaches it (~0.9 px/pass at radius 1, ~1.5–1.8 at radius 2); a precise FWHM-% guarantee is method A's job (T1-7).
- [x] **T1-7 Reduction A (resynthesis, default):** per star — local background (annulus + inpaint), PSF-scaled re-add at factor r, color from the unclipped core annulus, saturated-star protection.
  **AC = FR-5 full:** achieved median FWHM reduction within ±10 % of requested; star count preserved for r > 0; no ringing/dark halos on golden diffs; outside-mask bit-identity; deterministic.
  **Done** 2026-07-16. 18 tests (12 unit + 6 AC), all five FR-5 bars met on the real `basic-5k` through the full detect → mask → gate → resynthesis pipeline: **FWHM ratio 0.537 at requested r=0.5 (+7.4 %, within ±10 %)**; **star count preserved** (98.9 % of centroids keep ≥70 % of peak); **no sub-background dark halo**; **outside-gate bit-identity on `basic-5k` and `nightscape-fg`**; **deterministic** with a pinned golden hash (teeth verified via a `core_k` perturbation). Method rebuilds each star by radial resampling of its own pixels (new(d)=old(d/r) ⇒ new_fwhm = r·old_fwhm, peak preserved), not a model fit — no +14 % Moffat mismatch, and bilinear cannot ring (D-038). `r=0` is the starless path (FR-7 building block). Documented limits: a sampling floor near Nyquist (r=0.4 reaches +10.5 %), and a flat inpaint fill that a real wide-field image may want noise-matched (real-corpus refinement, D-027).
- [x] **T1-8 `starkit-cli`:** `process` (versioned preset JSON), `inspect` (catalog dump), folder batch with per-image JSON report, stable exit-code table, skip-on-error.
  **AC = FR-8:** 100-image batch unattended; re-run hash-identical; one poisoned file is reported and skipped without aborting the batch.
  **Done** 2026-07-16. 15 tests (9 unit + 6 AC), all three FR-8 bars proved **through the compiled binary** (so exit codes are tested, not just library logic): a 100-image batch runs unattended, a re-run is byte-identical, and one poisoned file is reported + skipped without aborting (exit **7 BATCH_PARTIAL**). `process` runs the full decode → detect → mask → gate → reduce → encode pipeline per channel (colour survives), ICC/EXIF preserved, output atomic; `inspect` dumps a schema-v1 measured catalog; `batch` writes one report per image. Choices in **D-039**: preset schema tagged and all-defaults-valid (`{}` is valid; foreign schema rejected; defaults kept in lockstep with core by a test), stable exit-code table (documented in `starkit-cli/CLAUDE.md`), and **no wall clock in any hashed output** — reports carry no timing and batch inputs are sorted, so re-runs are byte-identical.
  **Note:** FR-8's AC is fully met on synthetic images. The broader G1 photographer sign-off still needs the real corpus (D-027).
- [x] **T1-9 Bench harness + baselines:** synthetic 61 MP variant; record full-pipeline timing (target ≤ 10 s, hard cap 20 s on 8 cores).
  **AC:** baseline JSON committed; regression check wired into `ci.sh`.
  **Done** 2026-07-16. `basic-61mp` variant added (9568×6376, 18 166 stars, generated on demand, never hash-pinned). Measured through the real `process_image`: **compute 8.3 s median-of-3, single-threaded — within the 10 s target and well under the 20 s hard cap** (decode 0.6 s, encode 0.8 s). Baseline committed at `fixtures/expected/bench/basic-61mp.json`; `starkit bench` times the exact `process` pipeline and gates on the hard cap (machine-independent), wired into `ci.sh --full`. **Rayon deliberately not added** (D-040): the budget is met single-threaded, and parallelising the path whose determinism is a release invariant would spend risk for headroom not needed. Detect is the bottleneck (~4.9 s of the 8.3). The bench gate is the 20 s cap; the 8.3 s reference and 10 s target are advisory (a timing cannot be hash-pinned — same cross-machine reasoning as D-012).
